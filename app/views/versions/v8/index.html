Overview

Changes

  • Changed Death Arrears Payee flow to include a heirachy of payees
  • Changed BSP start page to include a brief summary and go there straight from the notification
  • Add spouse and deceased details page, rather than tabs
  • Add relationship autocomplete
  • Move BSP application fields to the BSP process
  • Ask if caller is pensionable age if between 63-65

Hypotheses

  • To remove the need to capture potentially 3 additional person's details, we use the heirachy to determine who the DAP is
  • After trouble with the cards and task list approach we are attempting to take the user straight to the page for clarity
  • The tabs pages we had ended up with one or two fields per tab and were being conditionally shown, the need of tabs appears to be gone
  • In order to get accurate values and help with the possible choices we provided an autocomplete field
  • Previously we took data (NINo, sex, address) for the purpose of the BSP application before determining eligibility, now we only ask the questions to determine eligibility
  • Rather than ask a potentially sensitive question (biological sex), we ask if they're pensionable age if they're within pension boundaries (63-65)
